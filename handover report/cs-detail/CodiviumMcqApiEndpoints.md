# CodiviumMcqApiEndpoints.cs — Detail Reference

## What it is
HTTP endpoint stubs for the MCQ quiz pages (mcq-parent.html and mcq-quiz.html).
Three routes. Delegates to backend question database (not yet implemented).

## Registration
```csharp
CodiviumMcqApiEndpoints.Map(IEndpointRouteBuilder app)
```

## GET /api/mcq/categories

### Input
JWT Bearer token (no query params)

### Output (200 OK)
```json
{ "categories": [ { "id": "language-basics", "displayName": "Language Basics" } ] }
```

### Notes
- Returns stable category list from question database
- May be cached aggressively (Cache-Control: max-age=3600)
- Used by: mcq-parent.html (loadCategories)

## GET /api/mcq/questions

### Input query params
| Param | Type | Required |
|---|---|---|
| categories | comma-separated strings | Yes |
| difficulty | basic/intermediate/advanced | Yes |
| count | int 1–50 | Yes |
| skipCorrect | bool | No |

### Output (200 OK)
```json
{ "sessionId": "UUID-v4", "questions": [ { "id", "category", "difficulty",
  "question", "options"[6], "correctIndices"[], "explanation", "nanoTutorial" } ] }
```
Server shuffles both questions array and each question's options before returning.
correctIndices index into the already-shuffled options.

### Notes
- Used by: mcq-quiz.html (fetchQuestions)
- sessionId is UUID v4 generated by server, returned in POST /api/mcq/results

## POST /api/mcq/results

### Input body
Full session result — sessionId, settings, timestamps, per-question answers[].
See contracts/mcq-quiz-api.contract.md §3 for complete schema.

### Output
202 Accepted { "recorded": true }
409 Conflict { "recorded": true } if sessionId already recorded (idempotent)

### What the backend must do
- Update per-user correct-question set
- Create one McqAttempt record per category represented in answers[]
- Validate correctIndices against database record

## Limitations
- All three endpoints are stubs — question database not yet implemented
- skipCorrect requires per-user correct-question tracking in the database
