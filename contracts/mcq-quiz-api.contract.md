# MCQ Quiz API Contract

**Version:** 1.0.0  
**Status:** Specification — not yet implemented  
**Related files:** `assets/components/mcq/mcq-parent.js`, `assets/components/mcq/mcq-quiz.js`, `backend/CodiviumMcq.cs`

---

## Overview

Three endpoints support the MCQ pages. They are designed to be independent — each can be
implemented and tested separately.

| Endpoint | Page | Purpose |
|---|---|---|
| `GET /api/mcq/categories` | `mcq-parent.html` | List of filterable categories |
| `GET /api/mcq/questions` | `mcq-quiz.html` | Filtered, shuffled question set |
| `POST /api/mcq/results` | `mcq-quiz.html` | Session result + per-question analytics |

All endpoints require an authenticated session. The user identity is resolved from the
session cookie/token — no explicit `userId` parameter on requests.

---

## 1. `GET /api/mcq/categories`

Returns the list of quiz categories available on the platform. Used by `mcq-parent.html`
to populate the category filter UI.

### Request

No body. No query parameters.

### Response `200 OK`

```json
{
  "categories": [
    { "id": "data-types",        "displayName": "Data Types & Data Structures" },
    { "id": "regular-expressions","displayName": "Regular Expressions" },
    { "id": "functions",          "displayName": "Functions" },
    { "id": "performance",        "displayName": "Performance" },
    { "id": "language-basics",    "displayName": "Language Basics" },
    { "id": "object-orientation", "displayName": "Object Orientation" },
    { "id": "builtins",           "displayName": "Builtins" }
  ]
}
```

### Notes

- Categories are drawn from `config/scoring-config.v1.json` §`categories.available_by_track`.
- The frontend stores `category.displayName` in the settings object that travels to the
  quiz page. The quiz page sends `displayName` values to `GET /api/mcq/questions`.
  If the backend uses IDs internally, it must accept displayName as an alias.
- Response is stable per platform version — may be cached aggressively.
- **No `subCategoryOf` field is required here.** The `SubCategory` field exists on individual
  exercises, not on category list entries. The adaptive recommendation engine derives
  subcategory relationships from exercise history server-side. Stretch recommendations arrive
  pre-computed via `GET /api/user/adaptive-state` — the MCQ category list plays no part in
  subcategory logic.

### Error responses

| Code | Meaning |
|---|---|
| `401` | Unauthenticated — redirect to login |
| `500` | Server error — frontend falls back to hardcoded list |

## 2. `GET /api/mcq/questions`

Returns a filtered, shuffled set of questions for a quiz session. Generates a stable
`sessionId` for the session. Called once when `mcq-quiz.html` loads.

### Request query parameters

| Parameter | Type | Required | Notes |
|---|---|---|---|
| `categories` | comma-separated strings | ✅ | Category displayNames or IDs (see §1 note) |
| `difficulty` | `basic`\|`intermediate`\|`advanced` | ✅ | |
| `count` | int (1–50) | ✅ | Max questions to return |
| `skipCorrect` | bool (`true`/`false`) | ❌ | Exclude previously correct questions. Default false |

### Response `200 OK`

```json
{
  "sessionId": "3f2a8c1d-9e4b-47a2-b610-0a1c2d3e4f5a",
  "questions": [
    {
      "id": "q_lang_basics_neg_idx_001",
      "category": "Language Basics",
      "difficulty": "basic",
      "question": "What does this print?\n\n```python\nx = [10, 20, 30]\nprint(x[-1])\n```",
      "options": [
        "10",
        "20",
        "30",
        "IndexError",
        "None",
        "SyntaxError"
      ],
      "correctIndices": [2],
      "explanation": "Negative indexing counts from the end of the sequence. x[-1] is the last element, which is 30.",
      "nanoTutorial": "## Negative Indexing\n\n```python\nmy_list = [10, 20, 30]\nmy_list[-1]  # 30 (last)\nmy_list[-2]  # 20\n```\n\nWorks for lists, tuples, and strings."
    }
  ]
}
```

### Question object schema

| Field | Type | Notes |
|---|---|---|
| `id` | string | Stable question identifier for database keying |
| `category` | string | Display name matching categories list |
| `difficulty` | `basic`\|`intermediate`\|`advanced` | |
| `question` | string | Supports fenced code blocks (``` python ``` ) and inline code (backtick) |
| `options` | string[6] | Always exactly 6 options. Same code support as `question` |
| `correctIndices` | number[] | Indices into `options` that are correct. Multiple = multi-select question |
| `explanation` | string | Shown in results review. Same code support |
| `nanoTutorial` | string \| null | Optional tutorial content. `null` = no Tutorial button shown |

### Notes

- Server shuffles questions and options before returning — the client does not shuffle.
- `correctIndices` are indices into the already-shuffled `options` array.
- If fewer questions match the filters than `count`, return all matching questions.
- `sessionId` is a UUID v4 generated by the server. The client stores it and returns it
  in `POST /api/mcq/results`.

### Error responses

| Code | Meaning |
|---|---|
| `400` | Invalid parameters — response body contains `{ "error": "message" }` |
| `401` | Unauthenticated |
| `404` | No questions match filters — `{ "questions": [], "sessionId": "..." }` |
| `500` | Server error — frontend shows error and Back to Setup |

## 3. `POST /api/mcq/results`

Records the complete quiz session result and all per-question analytics. Called once when
the summary page renders. Fire-and-forget — the client does not block on this response.

### Request body

```json
{
  "sessionId": "3f2a8c1d-9e4b-47a2-b610-0a1c2d3e4f5a",
  "settings": {
    "categories": ["Language Basics", "Functions"],
    "difficulty": "basic",
    "questionCount": 10,
    "skipCorrect": false
  },
  "startedAt": "2026-03-18T14:00:00.000Z",
  "completedAt": "2026-03-18T14:12:43.000Z",
  "minutesSpent": 12.7,
  "totalQuestions": 10,
  "correct": 7,
  "peeked": 1,
  "scorePercent": 70.0,
  "answers": [
    {
      "questionId": "q_lang_basics_neg_idx_001",
      "category": "Language Basics",
      "difficulty": "basic",
      "selectedIndices": [2],
      "correctIndices": [2],
      "isCorrect": true,
      "isPeek": false,
      "tutorialViewed": false,
      "submittedAt": "2026-03-18T14:01:22.000Z",
      "questionIndex": 0
    }
  ]
}
```

### Request body fields

**Session-level fields**

| Field | Type | Required | Notes |
|---|---|---|---|
| `sessionId` | string (UUID) | ✅ | Returned by `GET /api/mcq/questions` |
| `settings` | object | ✅ | Filter settings used for this quiz |
| `settings.categories` | string[] | ✅ | |
| `settings.difficulty` | string | ✅ | |
| `settings.questionCount` | int | ✅ | |
| `settings.skipCorrect` | bool | ✅ | |
| `startedAt` | ISO 8601 UTC | ✅ | When quiz page loaded |
| `completedAt` | ISO 8601 UTC | ✅ | When summary was triggered |
| `minutesSpent` | number | ✅ | `(completedAt - startedAt)` in minutes |
| `totalQuestions` | int | ✅ | |
| `correct` | int | ✅ | Questions answered correctly (not peeked) |
| `peeked` | int | ✅ | Questions where answer was revealed |
| `scorePercent` | number (0–100) | ✅ | `correct / totalQuestions * 100` |
| `selectedRecommendationType` | string \| null | ✅ | Adaptive recommendation type that led to this session (e.g. `weakness_spotlight`, `re_entry`). Null if not launched from adaptive practice page. |
| `answers` | array | ✅ | One entry per question attempted |

**Per-question answer fields** (`answers[]`)

| Field | Type | Required | Notes |
|---|---|---|---|
| `questionId` | string | ✅ | Stable question ID from `GET /api/mcq/questions` |
| `category` | string | ✅ | For denormalized analytics queries |
| `difficulty` | string | ✅ | For denormalized analytics queries |
| `selectedIndices` | number[] | ✅ | Which options the user chose. Empty array if peeked without selecting |
| `correctIndices` | number[] | ✅ | Echo of what the server sent — for server-side validation |
| `isCorrect` | bool | ✅ | |
| `isPeek` | bool | ✅ | True = user clicked "View Answer" |
| `tutorialViewed` | bool | ✅ | True = user opened the Tutorial panel for this question |
| `submittedAt` | ISO 8601 UTC | ✅ | Timestamp of submission or peek confirmation |
| `questionIndex` | int | ✅ | 0-based position in the quiz (for ordering) |
| `responseTimeSeconds` | int \| null | ✅ | Seconds from question render to submission. Null if timing unavailable. Used for session quality (Foundation/Consolidation/Proficient/Fluent) calculation. |

### Response `202 Accepted`

```json
{ "recorded": true }
```

### Notes

- The server should validate that `correctIndices` in the payload matches its own
  database record for each `questionId`. Discard and flag any answer where they differ.
- The `answers` array maps to `McqAttempt` for the scoring engine:
  `sessionId` → `QuizId`, primary category → `CategoryId`, `scorePercent` → `ScorePercent`,
  `completedAt` → `CompletedAtUtc`.
- Multi-category sessions: create one `McqAttempt` entry per distinct category represented
  in the `answers` array. Score each category separately using only the questions for that
  category.
- After recording, update the user's distinct-correct-question count (`UniqueCorrectMcqAllTime`
  in the raw data payload) by adding any `questionId` values where `isCorrect = true` and
  the question was not already in the user's correct set.

### Error responses

| Code | Meaning |
|---|---|
| `400` | Invalid body — validation errors in response |
| `401` | Unauthenticated |
| `409` | `sessionId` already recorded — idempotent, respond `{ "recorded": true }` |
| `500` | Server error — client logs warning, summary still renders |

## Backend DTOs (C# reference)

```csharp
// McqSessionResult: maps POST /api/mcq/results body
public sealed record McqSessionResult
{
    public required string SessionId { get; init; }
    public required McqSessionSettings Settings { get; init; }
    public required DateTime StartedAtUtc { get; init; }
    public required DateTime CompletedAtUtc { get; init; }
    public required double MinutesSpent { get; init; }
    public required int TotalQuestions { get; init; }
    public required int Correct { get; init; }
    public required int Peeked { get; init; }
    public required double ScorePercent { get; init; }
    public required IReadOnlyList<McqQuestionAnswer> Answers { get; init; }
}

public sealed record McqSessionSettings
{
    public required IReadOnlyList<string> Categories { get; init; }
    public required string Difficulty { get; init; }
    public required int QuestionCount { get; init; }
    public required bool SkipCorrect { get; init; }
}

public sealed record McqQuestionAnswer
{
    public required string QuestionId { get; init; }
    public required string Category { get; init; }
    public required string Difficulty { get; init; }
    public required IReadOnlyList<int> SelectedIndices { get; init; }
    public required IReadOnlyList<int> CorrectIndices { get; init; }
    public required bool IsCorrect { get; init; }
    public required bool IsPeek { get; init; }
    public required bool TutorialViewed { get; init; }
    public required DateTime SubmittedAt { get; init; }
    public required int QuestionIndex { get; init; }
}
```

## Mapping to existing `McqAttempt` (dashboard scoring engine)

The `McqSessionResult` feeds the existing `McqAttempt`-based scoring pipeline:

| `McqSessionResult` field | → `McqAttempt` field |
|---|---|
| `SessionId` | → `QuizId` |
| `Settings.Difficulty` | → `Difficulty` |
| `ScorePercent` | → `ScorePercent` |
| `CompletedAtUtc` | → `CompletedAtUtc` |
| `TotalQuestions` | → `TotalQuestions` |
| `Correct` | → `Correct` |
| `MinutesSpent` | → `MinutesSpent` |
| Per-category from `Answers[]` | → `CategoryId` (one `McqAttempt` per distinct category) |
